# CLAUDE.md — `examples/rust-by-example/`

A from-scratch, **incremental, hand-written** port of upstream [rust-by-example](https://github.com/rust-lang/rust-by-example) to Typst documents that evaluate end-to-end through `evcxr-typst`.

## How this differs from issue #20

GitHub issue #20 (and `docs/tracks/rust-by-example-port.md`, T-B01..T-B06) describes a *deterministic mechanical port* driven by a Rust binary in `tools/rbe-port/`. This directory is the **alternative path**: chapters are written by hand, one at a time, with the goal of *learning the tool through use* and capturing that learning as journal entries, tutorials, and tests.

Both paths can coexist. If/when `tools/rbe-port/` ships, it can backfill chapters not yet covered here, or replace this directory wholesale. Until then, this is where porting happens.

## Directory shape

```
examples/rust-by-example/
├── CLAUDE.md         (this file)
├── README.md         (human-facing intro + chapter index)
├── NOTICES.md        (license attribution to upstream rust-by-example)
├── main.typ          (entry; #include's chapter files in SUMMARY order)
└── <chapter>.typ     (one per ported chapter, mirroring upstream paths)
```

Chapter `.typ` files mirror upstream paths (e.g. `hello.typ`, `primitives/literals.typ`). The top-level `main.typ` `#include`s them in the order of upstream's `SUMMARY.md` so the whole book renders as one document.

## Invariants for chapter files

- Every chapter must `#import "../../packages/evcxr/lib.typ" as evcxr` (relative — same as `examples/hello/`).
- Bare `typst compile main.typ` (or any chapter standalone) **must** succeed (D-004). Verify before committing.
- `evcxr-typst run --allow-eval` on `main.typ` must produce captured stdout / sidecars matching upstream's expected output for evaluable snippets.
- Each chapter's first non-import line is a comment of the form `// Adapted from rust-by-example/<src-path>.md (see NOTICES.md).` (Auto-generated by the porter when issue #20 ships; for hand-written chapters here, write it yourself.)

## Fidelity vs. evcxr's evaluation model

Upstream wraps almost every snippet in `fn main() { ... }`. evcxr executes top-level statements and would *define* `main` without calling it. Until `rust-main(...)` ships (T-B01, D-022), hand-written chapters have a choice:

1. **Inline body, drop the wrapper.** Most natural for evcxr; loses the upstream "this is `main`" pedagogical framing.
2. **Show the wrapper, append a synthetic `main();` call to the visible source.** Fidelity to upstream's *visible* source; the trailing call is honest about the eval model.
3. **Wait for `rust-main(...)`.** Cleanest long-term, but blocks chapters today.

Pick *one* per chapter, document the choice in the journal entry, and stay consistent. The current default (until T-B01 ships) is **option 1**: drop the wrapper. See `journal/2026-05-09-001-hello.md`.

**Pick option 2 instead** when the upstream `fn main()` body declares `let` bindings or items (`struct` / `enum` / `fn`) that should *not* leak across chapters. Inlining them as top-level evcxr statements would commit the bindings / items to subsequent chapters' scope. Wrapping them in `fn main() { ... }` and calling `main();` keeps them function-local. See `journal/2026-05-10-001-print.md` for the worked example (bindings `number`, `width` plus a local `struct Structure(i32)`).

## When you finish a chapter

1. Verify `typst compile --root . main.typ` succeeds (fallback path).
2. Verify `cargo run -p evcxr-typst -- run --allow-eval --root . main.typ` produces the expected sidecars next to the entry file.
3. Update `README.md`'s chapter index.
4. Write a `journal/YYYY-MM-DD-NNN-<chapter>.md` entry.
5. If something is now stable enough to teach, draft a `docs/tutorial/<topic>.md`.
6. If a behaviour is worth pinning, add a test under `crates/evcxr-typst/tests/rbe_*.rs`.

## What does NOT belong here

- Logic in the chapter file beyond `#evcxr.rust(...)` etc. — keep chapters as faithful as possible to upstream prose plus snippet markers.
- Pre-created empty subdirs for chapters not yet ported. Add as you go.
- Per-chapter sidecar fixtures. The cache directory `.evcxr-typst-cache/` is gitignored.
